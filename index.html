Conceptual questions

  1. What is DOM?
    2. -DOM stands for DOCUMENT OBJECT MODEL. It is how web browsers represent internally.

    i)Document - we can think of document as any piece of papers that represents a certain information like books , articles, etc. In case of web developers a document is a name for web page

    ii)Object - The things which are on web pages (Documents) are called objects sometimes called elements or nodes. examples:-
    ----Content:- The most obvious objects on a web page are the content. These can include words, videos, images, and more.
    ----Elements:- These include divs, containers, and sections. You may not see these elements, but you see how they affect the visible elements because they organize those elements on the web page.
    ----Attributes:- Every element on a web page has attributes.These include classes, styles, and sizes for example. These are objects in the DOM, but they dont have features like contents and elements.

    iii)Model - A model is a representation of something, and it helps us understand how something is put together. There are models for many things that need to be universally understood, analyzed, and used.

    So what exactly is a DOM?
    The DOM is represented as a type of data structure called a tree. Every object in the DOM is hierarchically under another object, and any object can have multiple children but only one parent.

  2. What is an API?
    3. API stands for Application Programming Interface. API is a set of routine, protocol and tools for building software applications. It basically connects two diffrent set of programs or data which gives an interaction between them. Basically connecting server-side to client side.

  3. What do you mean by semantic HTML ?
    4. We can structure a HTML page anyhow without caring about its meaning the output will be same there will be no major changes in it but at the same time the meaning of the page might be misaligned for e.g:- 
    <H1>FOO<H1> <p>FOO<p>
    We can use both H1 and P tags for heading by increasing the size of text of p tag but this might result in wrong meaning, We can use label tag for better understanding for visually impared persons. The tag will be recognized by the browser and will read it's content. for e.g:- 
    <label for="FOO">FOO<H1>

  4. What is the diffrence between inline and inline-block?
    5. First lets understand what is inline and block element.
    Inline elements:-There is no line break around this element if you place another line ahead or behind of it, the element or the line will continue from there only. In short the element just takes it's own content's space.
    Block elements:-Block element does'nt let another line or tag interupt with it's spacing. In short it applies a line break after and before the element.
    Inline-Block elements:- By default we can't alter or reset the margin or padding for up and bottom of the inline element that's where Inline-Block comes to use we can alter inline-block's margin and padding.

  6. What are the ways of linking stylesheet to HTML file ?
    7. We can link a css file in HTML in three ways:-
    i) inline css :- by using the style attribute inside HTML elements
    ii) internal css :- by using a <style> element in the <head> section
    iii) external css :- by using a <link> element to link to an external CSS file

  7. Consequences of placing script tag inside head ?
    8. The script tag should be placed above the closing body tag after writting all the HTML code.
